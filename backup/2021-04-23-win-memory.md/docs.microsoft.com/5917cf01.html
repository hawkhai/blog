工作集Working Set

  • 2018/05/31
    • m
    • o

本文内容

进程的工作集是当前驻留在物理内存中的进程的虚拟地址空间中的一组页。The working set of a process is the set of pages in the virtual address space of the process that are currently resident in physical memory. 工作集只包含可分页的内存分配;不可分页内存分配，如 (AWE) 或大型页面分配的地址窗口扩展，工作集中不包括此类分配。The working set contains only pageable memory allocations; nonpageable memory allocations such as Address Windowing Extensions (AWE) or large page allocations are not included in the working set.

当进程引用当前不在其工作集中的可分页内存时，将出现 页错误 。When a process references pageable memory that is not currently in its working set, a page fault occurs. 系统页错误处理程序尝试解决页错误，如果成功，则将该页添加到工作集。The system page fault handler attempts to resolve the page fault and, if it succeeds, the page is added to the working set. (访问 AWE 或大型页面分配从不导致页错误，因为这些分配不可分页。 ) (Accessing AWE or large page allocations never causes a page fault, because these allocations are not pageable .)

必须通过从页面的 后备存储（即系统分页文件或该进程创建的内存映射文件）读取页面内容来解决 硬页错误。A hard page fault must be resolved by reading page contents from the page's backing store, which is either the system paging file or a memory-mapped file created by the process. 无需访问后备存储即可解决 软页错误 。A soft page fault can be resolved without accessing the backing store. 出现软页错误的时间：A soft page fault occurs when:

  • 该页处于某个其他进程的工作集，因此它已驻留在内存中。The page is in the working set of some other process, so it is already resident in memory.
  • 该页处于转换中，因为它已从使用页面但尚未改变用途的所有进程的工作集中删除，或者已驻留在内存管理器预提取操作的结果中。The page is in transition, because it either has been removed from the working sets of all processes that were using the page and has not yet been repurposed, or it is already resident as a result of a memory manager prefetch operation.
  • 进程首次引用分配的虚拟页时 (有时称为 " 请求-零错误) "。A process references an allocated virtual page for the first time (sometimes called a demand-zero fault).

可从进程工作集删除页面，作为以下操作的结果：Pages can be removed from a process working set as a result of the following actions:

  • 此进程通过调用 SetProcessWorkingSetSize、 SetProcessWorkingSetSizeEx 或 EmptyWorkingSet 函数来减少或清空工作集。The process reduces or empties the working set by calling the SetProcessWorkingSetSize, SetProcessWorkingSetSizeEx or EmptyWorkingSet function.
  • 此过程对未锁定的内存范围调用 VirtualUnlock 函数。The process calls the VirtualUnlock function on a memory range that is not locked.
  • 进程使用 UnmapViewOfFile 函数 messagebox 取消文件的映射视图。The process unmaps a mapped view of a file using the UnmapViewOfFile function.
  • 内存管理器将从工作集中修整页面，以创建更多可用内存。The memory manager trims pages from the working set to create more available memory.
  • 内存管理器必须从工作集中删除一个页面，以便为新页面腾出空间，例如，因为工作集的最大大小为) 的 (。The memory manager must remove a page from the working set to make room for a new page (for example, because the working set is at its maximum size).

如果有多个进程共享某个页面，则从一个进程的工作集中删除该页面不会影响其他进程。If several processes share a page, removing the page from the working set of one process does not affect other processes. 从使用它的所有进程的工作集中删除某一页后，此页将成为一个 转换页。After a page is removed from the working sets of all processes that were using it, the page becomes a transition page. 转换页面将一直在 RAM 中进行缓存，直到某个进程或重新用途的页再次引用该页 (例如，用零填充，并将其提供给另一个处理) 。Transition pages remain cached in RAM until the page is either referenced again by some process or repurposed (for example, filled with zeros and given to another process). 如果在上次将转换页面写入磁盘后已对其进行了修改 (也就是说，如果页面是 "脏" ) ，则必须将页面写入其后备存储，然后才能改变其用途。If a transition page has been modified since it was last written to disk (that is, if the page is "dirty"), then the page must be written to its backing store before it can be repurposed. 此类页面可用时，系统可能会立即开始将脏转换页面写入其后备存储。The system may start writing dirty transition pages to their backing store as soon as such pages become available.

每个进程都具有最小和最大工作集大小，这会影响进程的虚拟内存分页行为。Each process has a minimum and maximum working set size that affect the virtual memory paging behavior of the process. 若要获取指定进程的工作集的当前大小，请使用 GetProcessMemoryInfo 函数。To obtain the current size of the working set of a specified process, use the GetProcessMemoryInfo function. 若要获取或更改最小和最大工作集大小，请使用 GetProcessWorkingSetSizeEx 和 SetProcessWorkingSetSizeEx 函数。To obtain or change the minimum and maximum working set sizes, use the GetProcessWorkingSetSizeEx and SetProcessWorkingSetSizeEx functions.

"进程状态" 应用程序编程接口 (PSAPI.DLL) 提供了许多返回有关进程工作集的详细信息的函数。The process status application programming interface (PSAPI) provides a number of functions that return detailed information about the working set of a process. 有关详细信息，请参阅 工作集信息。For details, see Working Set Information.
